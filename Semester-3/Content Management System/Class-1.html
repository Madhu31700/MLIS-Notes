<h1> to <h6> — Headings

Code

<h1>This is header</h1>

Output

This is header

6 levels available: h1 (largest/most important) down to h6 (smallest). Only one h1 per page is best practice — it's the main title.

<hr> and <br>

Code

<hr>
This is paragraph<br>
This is <br> paragraph2

Output

This is paragraph
This is
paragraph2
<hr> = horizontal rule, draws a divider line (used to separate sections).
<br> = line break, forces text to a new line without starting a new paragraph. Both are self-closing / empty tags (no closing tag or content needed).

Text Formatting — <u> <b> and the invalid tag caught here

Code

today is <u><b>Tuesday</b></u>
today is <strang>Tuesday</strang>

Output

today is Tuesday
today is Tuesday
<u> underlines text, <b> makes it bold — tags can be nested (innermost applies last, outermost wraps everything).
⚠️ <strang> is not a real HTML tag (typo for <strong>). Browsers silently ignore unknown tags and just render the text inside — a good reminder that HTML doesn't "error out," it fails silently. Correct tag: <strong> (bold + semantic importance).

Lists — <ol> and <ul>

Code

<ol type="I">
  <li>Item1</li>
  <li>Item2</li>
</ol>

<ul>...</ul>              <!-- default: disc -->
<ul type="circle">...</ul>
<ul type="square">...</ul>

Output

  1. Item1
  2. Item2
  • Item1
  • Item2
  • Item1
  • Item2
  • Item1
  • Item2
<ol> = ordered list (numbered/lettered). type attribute controls the marker style:
  type="1" numbers (default) · type="A" A,B,C · type="a" a,b,c · type="I" Roman numerals · type="i" lowercase roman.
<ul> = unordered list (bullets). type values: disc (default, filled circle) · circle (hollow) · square.
Every item inside either list uses <li> (list item).

<a> — Anchor / Hyperlink

Code

<a href="https://google.com">Google</a>

Output

Google
href = the destination URL (required). The text between opening/closing tags is what's clickable. Useful extra attributes: target="_blank" (opens in new tab), title="..." (tooltip on hover).

<img> — Images

Code

<img src="logo.png" height="200"
     alt="Indian Statistical Institute" />

Output

src = image path/URL (required). height / width = size in pixels (set only one to keep aspect ratio). alt = fallback text shown if the image fails to load, and read aloud by screen readers — always include it. <img> is self-closing (/>).

<table> — Basic Structure

Code

<table border="1" cellpadding="10">
  <tr>
    <th>SN</th>
    <th>State</th>
  </tr>
  <tr>
    <td>1.</td>
    <td>Karnataka</td>
  </tr>
</table>

Output

SNState
1.Karnataka
<table> wraps the whole table. <tr> = table row. <th> = header cell (bold, centered by default). <td> = normal data cell.
border = thickness of the grid lines. cellpadding = space between a cell's border and its content. (Modern CSS usually replaces these attributes, but they still work.)

rowspan / colspan / width

Code

<tr>
  <td>6.</td>
  <td>Punjab</td>
  <td rowspan="2">Chandigarh</td>
</tr>
<tr>
  <td>7.</td>
  <td>Haryana</td>
</tr>

<td colspan="2">This is Header</td>
<td width="25%">Menu</td>

Output

6.PunjabChandigarh
7.Haryana
rowspan="2" on a cell makes it stretch vertically across 2 rows — so the next row must have one fewer <td> (that cell is "absorbed").
colspan="2" makes a cell stretch horizontally across 2 columns — used for a full-width header/footer row.
width="25%" fixes a column's width as a percentage of the table.

Nested Tables

Code

<table>
  ...outer rows...
  <table>   <!-- table inside a cell -->
    <tr><td colspan="2">Header</td></tr>
    <tr>
      <td width="25%">Menu</td>
      <td>Content</td>
    </tr>
    <tr><td colspan="2">Footer</td></tr>
  </table>
</table>

Output

This is Header
MenuContent
This is Footer
A <table> can be placed inside a cell of another table to build layouts (e.g. a mini header/menu/content/footer block within one row). Common in old-style page layouts before CSS Grid/Flexbox.
Correction: in the uploaded file the inner <table> was placed directly as a child of <tr>, not inside a <td> — technically invalid nesting (a nested table should sit inside a <td>). Browsers auto-correct it, but write it properly as: <td><table>...</table></td>.
